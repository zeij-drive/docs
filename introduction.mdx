# deobf-all 简介

**deobf-all** 是一个 Agent Skill，充当反混淆工作流的统一调度入口。一条命令 `npx skills add zeij-drive/deobf --skill deobf-all -g -y` 即可全局安装，调用 `/deobf-all` 后它会**先分类、再按需加载**相关子 skill，避免一次性拉起全部 26 个子 skill 浪费上下文。

<Steps>
  **分类（Triage）**
  
  检查目标文件，识别类型和混淆特征，产出分类摘要

  **加载 P0**
  
  只加载 1–2 个核心 skill（原生二进制 → `code-obfuscation-deobfuscation`，JavaScript → `ast-deobfuscation`）

  **按需加载 P1/P2**
  
  根据路由结果，仅加载当前目标需要的辅助 skill（如 VM 分析、反调试绕过等）

  **反混淆 → 验证**
  
  执行反混淆工作流，确认输出正确性
</Steps>

## 为什么选择 deobf-all？

<CardGroup cols={2}>
  <Card title="统一入口" icon="bolt">
    不必记住每个 skill 的名称。一条命令 `npx skills add zeij-drive/deobf --skill deobf-all -g -y` 全局安装，一个命令 `/deobf-all` 调用，自动完成全部调度。
  </Card>
  <Card title="按需加载" icon="route">
    先分类再加载，仅加载目标相关的子 skill，节省上下文空间和工具调用次数。每次调用不超过 5 次 `read_skill`。
  </Card>
  <Card title="全场景覆盖" icon="shield-check">
    从 VMProtect 到 obfuscator.io，从 OLLVM 到自定义 VM — 覆盖主流混淆与保护方案。
  </Card>
  <Card title="可扩展" icon="layers">
    架构设计为可扩展：新增子 skill 只需更新调度器的路由表。
  </Card>
</CardGroup>
