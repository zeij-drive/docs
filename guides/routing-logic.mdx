# 路由逻辑

deobf-all 内置路由矩阵，根据分类结果**仅加载目标需要的子 skill**，空单元格的 skill 不会被加载。这样可以保持每次调用不超过 5 次 `read_skill`。

## 路由矩阵

基于分类结果，`read_skill` **仅**加载标记 ✓ 的 skill：

| 目标 | P0 核心反混淆 | P0 AST 反混淆 | P1 VM 逆向 | P1 反调试 | P1 符号执行 | P2 保护绕过 | P2 CTF | P2 反逆向 | P2 深度分析 |
|------|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
| 原生二进制（通用） | ✓ | | | | | | | | |
| 原生 + VM 保护 | ✓ | | ✓ | | | | | | |
| 原生 + VM + 反调试 | ✓ | | ✓ | ✓ | | | | | |
| 原生 + CFF（OLLVM） | ✓ | | | | ✓ | | | | |
| 原生 + 加壳 + 反逆向 | ✓ | | | | | ✓ | | ✓ | |
| JavaScript（任意） | | ✓ | | | | | | | |
| JS + 重度混淆 | ✓ | ✓ | | | | | | | |
| DotNet/Java/Python 字节码 | ✓ | | ✓ | | ✓ | | | | |
| CTF 逆向挑战 | ✓ | | | | | | ✓ | | ✓ |
| 未知 / 不确定 | | | | | | | | | ✓ |

**规则**：空单元格 = 不要加载该 skill。

## Skill 优先级

调度器为子 skill 定义了三级优先级：

| 优先级 | 含义 | 加载时机 | Skill |
|--------|------|----------|-------|
| **P0** | 核心 skill | 分类后立即加载 | `code-obfuscation-deobfuscation`、`ast-deobfuscation` |
| **P1** | 按需加载 | 根据路由矩阵标记 ✓ | `vm-and-bytecode-reverse`、`anti-debugging-techniques`、`symbolic-execution-tools` |
| **P2** | 辅助增强 | 仅在特殊场景加载 | `binary-protection-bypass`、`ctf-reverse`、`anti-reversing-techniques`、`deep-analysis` |

<Info>
`deep-analysis`（P2）是唯一可能在 P0 之前加载的 skill — 仅当目标类型完全无法确定时作为 Fallback 使用。
</Info>

## Fallback — 分类不确定时

如果检查目标后仍无法确定类型：

1. 先 `read_skill` 加载 `deep-analysis`（P2）
2. 遵循 deep-analysis 方法论对目标分类
3. 回到路由矩阵，根据分类结果加载对应 P0/P1 skill

详见 [工作原理](/guides/how-it-works)。
